6 - AI ตอบลูกค้าอัตโนมัติผ่าน Chatwoot
What & Why
ตอนนี้ Virtual Office เป็น advisory tool — Founder วาง text ใน VA แล้วได้คำแนะนำ แต่ยังต้องไปตอบลูกค้าเอง Task นี้ทำให้ AI ตอบแทน Founder ได้เลยเมื่อลูกค้าทัก DM มา (Facebook Messenger, Instagram, Line) ผ่าน Chatwoot inbox Founder ยังอยู่ใน loop — เห็น conversation ใน Chatwoot, override ได้ตลอดเวลา
Done looks like
* ลูกค้าทัก Facebook Page → Chatwoot รับ → webhook ส่งให้ AI → AI ตอบอัตโนมัติภายใน 30 วินาที
* AI ใช้ customer context (stage, history) เพื่อตอบให้ตรงจุด ไม่พูดซ้ำ
* ถ้า AI ไม่มั่นใจ (เช่น ลูกค้าถามราคาแบบ custom) → assign conversation ให้ Founder พร้อม draft ที่ AI เตรียมไว้
* Virtual Office แสดง live feed ของ conversations ที่ AI กำลังจัดการ
* Founder กด "Take over" ได้เพื่อ reply เอง
Out of scope
* Auto-close deals / รับเงินอัตโนมัติ (แค่คุยถึงขั้นตอนชำระเงิน)
* Voice / video calls
* TikTok (แยก task)
Steps
1. Chatwoot webhook receiver — สร้าง /webhook/chatwoot endpoint รับ incoming message events, verify HMAC signature, extract contact + conversation + message
2. AI reply pipeline — เมื่อรับ webhook → load lead context → route ไปยัง sales_assistant หรือ customer_success_agent → ส่ง reply กลับผ่าน Chatwoot API
3. Confidence gating — agent emit confidence score; ถ้าต่ำกว่า threshold → assign to human + แนบ draft ใน Chatwoot private note
4. Chatwoot ↔ Lead sync — สร้าง/อัพเดท Lead record จาก Chatwoot contact; เก็บ chatwoot_conversation_id เพื่อ link กัน
5. Virtual Office live feed UI — widget ใน VA แสดง active AI conversations แบบ real-time (SSE หรือ polling 5s)
Relevant files
* beauty_agent_system/app/chatwoot_client.py
* beauty_agent_system/app/routers/office.py
* beauty_agent_system/app/models.py
* beauty_agent_system/app/agents/sales_assistant.py
* beauty_agent_system/app/agents/customer_success_agent.py
* beauty_agent_system/app/config.py

#7 - Facebook comment monitoring & auto-DM prospecting
What & Why
แทนที่จะรอให้ลูกค้าทักมา ระบบจะไปสอดส่องโพสต์ใน Facebook Page ของ shop เอง ถ้าคนคอมเมนต์แบบที่บ่งชี้ความสนใจ (เช่น "ราคาเท่าไหร่", "สนใจ", "ทักมาได้เลย") → AI reply comment + ส่ง DM ทันที ช่วยให้ไม่พลาด hot lead ที่คอมเมนต์แล้วรอ
Done looks like
* ระบบ poll Facebook Page posts ทุก 5 นาที หาคอมเมนต์ใหม่
* AI classify comment: "buying signal" / "question" / "noise"
* ถ้าเป็น buying signal → post public reply (สั้นๆ เชิญ DM) + ส่ง DM ทันที
* ถ้าเป็น question → post public reply ตอบคำถามโดยตรง
* บันทึก lead ใน leads table ทันทีที่ DM ออก
* Founder เห็น log ว่า AI ทัก lead ไหนไปบ้างใน Virtual Office
Out of scope
* Monitor competitor pages (Facebook ToS ไม่อนุญาต scraping)
* Facebook Ads automation (ใช้ Facebook Ads Manager แทน)
* Instagram (ใช้ Chatwoot ซึ่งรองรับ Instagram อยู่แล้ว ไม่ต้องทำซ้ำ)
Steps
1. Facebook Graph API integration — setup Page access token, สร้าง facebook_client.pyสำหรับ list posts, list comments, post reply, send DM (via Messenger Send API)
2. Comment classifier — ใช้ lead_hunter agent classify comment text: buying_signal, question, noise; cache result ต่อ comment_id เพื่อไม่ classify ซ้ำ
3. Auto-reply & DM flow — logic ตัดสินใจ reply/DM; rate limit ไม่เกิน 20 DM/ชั่วโมงเพื่อหลีกเลี่ยง Facebook spam detection
4. Polling scheduler — APScheduler job ทำงานทุก 5 นาที; เก็บ last_checked_at per post ใน research_cache table
5. Prospecting dashboard UI — Virtual Office tab แสดง leads ที่ได้จาก Facebook: ชื่อ, comment, stage, วันที่ทัก
Relevant files
* beauty_agent_system/app/models.py
* beauty_agent_system/app/agents/lead_hunter.py
* beauty_agent_system/app/config.py
* beauty_agent_system/run.py

8 - TikTok comment monitoring & auto-reply
What & Why
TikTok เป็น top-of-funnel ที่ดีสำหรับ beauty business — คลิปสาธิต, before/after, review ถ้ามีคนคอมเมนต์สนใจใต้คลิป AI จะตอบเองทันที แล้วเชิญมา DM หรือ inbox เพื่อปิดการขาย
Done looks like
* ระบบดึง comments จาก TikTok videos ของ shop account ทุก 10 นาที
* AI classify และ reply comments ที่เป็น buying signal หรือ question
* ถ้าลูกค้า reply กลับ → AI ตอบต่อได้ (threaded)
* Lead ที่มาจาก TikTok บันทึกใน leads table พร้อม source tag
* Founder เห็น TikTok leads แยก tab ใน Virtual Office
Out of scope
* TikTok DM automation (TikTok API ไม่เปิด DM สำหรับ third-party ณ ตอนนี้ — ใช้การ reply comment + ชี้ไป Line/Facebook แทน)
* TikTok Ads (แยกต่างหาก)
* Upload / edit TikTok videos
Steps
1. TikTok for Business API integration — สมัคร TikTok for Business developer account, implement OAuth 2.0 flow, สร้าง tiktok_client.py สำหรับ list videos, list comments, post comment reply
2. Comment polling & deduplication — APScheduler job ทุก 10 นาที ดึง comments ใหม่ เก็บ processed comment_ids ใน research_cache เพื่อไม่ reply ซ้ำ
3. Classify & reply pipeline — ใช้ lead_hunter classify แล้ว AI สร้าง reply ที่เป็น natural ไม่ดู bot (ห้าม template ซ้ำทุก comment)
4. Cross-channel lead merge — ถ้า lead เดียวกันทัก Facebook และ comment TikTok → merge เป็น lead record เดียวโดย match ชื่อ / phone ถ้ามี
5. TikTok leads tab — Virtual Office แสดง leads จาก TikTok พร้อม link ไปยัง comment ต้นทาง
Relevant files
* beauty_agent_system/app/models.py
* beauty_agent_system/app/agents/lead_hunter.py
* beauty_agent_system/app/config.py
* beauty_agent_system/run.py
